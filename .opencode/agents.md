# Конфигурация агентов OpenCode

## Доступные Skills

| Skill | Файл | Назначение |
|-------|------|------------|
| **plan** | skills/plan.md | Создание планов изменений |
| **implement** | skills/implement.md | Реализация планов |
| **review** | skills/review.md | Проверка качества |
| **content** | skills/content.md | Написание контента |
| **deploy** | skills/deploy.md | Публикация на GitHub Pages |

## Типичные сценарии

### Добавление новой части презентации

```
1. /plan "Добавить часть 2 про результаты"
   → Planner создаёт план

2. [проверь docs/plan.md]

3. /implement
   → Executor создаёт файлы

4. /content "Напиши контент про результаты"
   → Content пишет текст

5. /review
   → Reviewer проверяет

6. /deploy
   → Deploy публикует
```

### Редактирование существующих слайдов

```
1. /content "Добавить слайд про метрики в part1"
   → Content создаёт слайд

2. /review
   → Reviewer проверяет

3. /deploy
   → Deploy публикует
```

### Быстрое обновление

```
/opencode "Добавить слайд с итогами и опубликуй"
→ Автоматически выберет нужные skills
```

## Автосохранение

После каждого изменения:
```bash
git add .
git commit -m "авто: описание изменения"
```

При готовности к публикации:
```bash
git push
```
