## 1. Проблема

### Контекст

Тестировщики сталкиваются с непонятными ошибками на UI и пингуют разработчиков починить это. Уходит много времени, так как с ходу непонятно где и что упало в распределенной системе.

### Задача агента

Выяснить что сломалось, дать анализ ошибки, тем самым ускорив исследования проблем на тест-стендах.

### Статистика

В среднем за 3 месяца тестировщики сталкиваются с **~120 ошибками**, затрачивая около **54 минут на каждую**.

---

## 2. Решение

### Процесс

1. Тестировщик воспроизводит ошибку
2. Данные об ошибке поступают ИИ агенту
3. Агент по MCP ходит в k8s и jaeger за информацией
4. Собирает логи и выдает анализ корня проблем
5. Снижает необходимость копаться в инфраструктуре

---

## 3. Архитектура

### Компоненты системы

| Компонент | Назначение |
|-----------|------------|
| UI (node.js) | Воспроизведение ошибки |
| Mongo | Хранение данных об ошибках |
| AlfaGen LLM | Qwen 32B / GPT OSS 120B |
| K8S tool | Доступ к логам кластера |
| Jaeger tool | Трассировка запросов |
| MCP | Протокол для инструментов |

### Принцип работы

Анализ ошибки в цикле, принятие решений о вызове инструментов MCP.

| Метрика | Значение |
|---------|----------|
